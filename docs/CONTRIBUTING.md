# Contributing

We welcome contributions to the **Urban Disruption Intelligence Engine**.

## Development Workflow
1. **Fork** the repository.
2. **Clone** it: `git clone https://github.com/fallofpheonix/UDIE`
3. **Branch**: `git checkout -b feature/your-feature`
4. **Develop**: Follow the modular style patterns in `docs/ARCHITECTURE.md`.
5. **Test**: Ensure any new feature includes basic unit tests or manual verification logs.
6. **Pull Request**: Submit your changes for review.

## Code Standards
- **Swift**: Follow standard Ray Wenderlich/Swift style guide. Use SwiftUI for all UI.
- **TypeScript**: Use strict mode. Prefer early returns and meaningful logging.
- **SQL**: Always use parameterized queries for PostGIS functions.
- **Documentation**: Update the relevant `.md` files in `docs/` if you change core logic.
