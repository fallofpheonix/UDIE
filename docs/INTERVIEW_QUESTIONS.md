# Interview Questions

## Project Understanding
1. What problem does UDIE solve, and how is success measured?
2. What are the strict in-scope and out-of-scope boundaries?
3. Which non-functional requirements are hardest to satisfy and why?

## Systems Design
1. Describe end-to-end data flow and trust boundaries.
2. Which invariants must never be violated?
3. What are top bottlenecks and mitigation strategy?

## Reliability and Operations
1. What are top three failure modes and recovery procedures?
2. How are SLO/SLA breaches detected and escalated?
3. What is rollback strategy for a bad release?

## Code Quality and Testing
1. Which tests gate deployment?
2. How is deterministic behavior validated?
3. Which metrics prove regression did not occur?

## Security
1. What threat model was assumed?
2. How are secrets managed and rotated?
3. What controls protect sensitive data at rest/in transit?

## Process and Collaboration
1. How are architecture changes approved?
2. How are risks tracked and retired?
3. Which decisions remain unresolved and why?
